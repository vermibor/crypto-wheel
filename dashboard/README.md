# Crypto Wheel — Trading Dashboard

Real-time monitoring dashboard for the Wheel Strategy Bot's BTC options trading performance.

## Overview

This Next.js dashboard visualizes performance data from the [Wheel Strategy Bot](../README.md), including:

- **Portfolio Performance** — Equity curve, cumulative P&L, and returns over time
- **Trade History** — Detailed log of all options trades (puts/calls sold, assignments, expirations)
- **Cash Flow Analysis** — Premium income, assignment costs, and net cash flow
- **Strategy Metrics** — Win rate, average premium, days in trade, and per-strategy breakdowns

## Data Flow

```
main.py (daily cron)
  → Executes strategies, exports CSVs
  → publish.py generates docs/data/dashboard.json
  → GitHub Actions copies data to dashboard/public/data/
  → Next.js static build → GitHub Pages
```

The dashboard reads a single `dashboard.json` file at runtime. This file is generated by `publish.py` from the bot's trade state and CSV exports, then copied into the dashboard during CI build.

## Local Development

```bash
# From the dashboard/ directory
npm install

# Copy sample data (or generate with publish.py)
mkdir -p public/data
cp ../docs/data/dashboard.json public/data/dashboard.json

# Start dev server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

> **Note:** The dashboard expects `public/data/dashboard.json` to exist. Without it, you'll see loading/error states. Run the bot or `publish.py` to generate this file.

## Production Build

```bash
npm run build
```

Outputs a fully static site to `out/` configured for the `/crypto-wheel/` base path on GitHub Pages.

## Deployment

Deployment is automated via GitHub Actions (`.github/workflows/deploy.yml`):

- **Triggers:** Push to `main` (changes in `docs/data/**` or `dashboard/**`), manual dispatch, daily at 10:30 UTC
- **Process:** Copies latest `dashboard.json` → installs deps → builds → deploys to GitHub Pages

Live at: `https://vermibor.github.io/crypto-wheel/`
